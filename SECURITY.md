# 🛡️ Security Policy

Thank you for helping improve the security of **SSRFHunter Pro**.

This project is developed for **authorized security assessments, security research, and educational purposes only.** Unauthorized testing of systems without explicit permission is prohibited.

---

## 📌 Supported Versions

| Version | Supported |
|:--------:|:---------:|
| Latest Release | ✅ Yes |
| Development Branch | ✅ Active |
| Older Versions | ❌ No |

---

## 🚨 Reporting a Security Vulnerability

If you discover a security vulnerability within **SSRFHunter Pro**, please report it responsibly.

**Please include:**
- A clear description of the issue
- Steps to reproduce
- Expected vs. actual behavior
- Impact assessment
- Proof of Concept (if applicable)
- Suggested remediation (optional)

Please **do not disclose vulnerabilities publicly** until they have been reviewed and addressed.

---

## ⚖️ Responsible Usage

SSRFHunter Pro must only be used against systems that you:

- ✅ Own
- ✅ Manage
- ✅ Have explicit written authorization to test

Users are solely responsible for ensuring their activities comply with applicable laws, regulations, and organizational policies.

---

## 🔒 Project Security Principles

The project is designed with security and responsible research in mind.

- Secure-by-default configuration
- No destructive functionality
- No persistence mechanisms
- No privilege escalation features
- No malware or offensive payloads
- Transparent logging and reporting
- Modular, auditable source code

---

## 🤝 Responsible Disclosure

We appreciate responsible security research and welcome reports that help improve the project while protecting users and the wider community.

Thank you for contributing to a safer open-source ecosystem.
